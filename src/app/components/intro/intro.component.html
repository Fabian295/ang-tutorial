Components

Angular has a component based structure, wraapped in at least one Module.

Having a modulair and component bassed structure comes with various benefits, but here are some of the major ones:

Reusability

Because you will work with modules built with several components and services, behaviours, directives, etc These building blocks can be used for multiple purposes or simply for the same purpose multiple times.

Maintainability

An application is now built in relatively small and multiple parts and subparts, instead of one or a couple of very large, inconvenient and obscure parts

Clean and good structured code

Angular is TypeScript based, which meanes it is easier to catch bugs, can do this before run time. Keeps you on track with return types, so you can spot bad syntax early in the game, at compilation time, before it is published.
Within Angular the library RXjs is also accessible which means wee have a lot of extra possibilities, like Observabless, Subscripion, many operators like : map, filter, etc ...

Ideally suited for collaboration

Generally speaking there are some agreements and "Best Practice" way's of witing the syntax. This leads to cleaner and generic code and thus easier and better understandable syntax for other developers, like your colleages
